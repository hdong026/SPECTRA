# KASA-ST

**K**olmogorov-**A**rnold **S**patiotemporal **A**ttention for traffic **S**eries **T**ime forecasting.

KASA-ST augments a spatiotemporal forecasting backbone with a **KAN (Kolmogorov–Arnold Network)** branch on the Prior channel. It models four input channels—traffic flow, time-of-day, day-of-week, and a data-driven **Prior**—to improve multi-step traffic forecasting.

---

## Features

- **Four-channel input**: Traffic flow (Flow) + Time-of-Day (TOD) + Day-of-Week (DOW) + **Prior** (generated by scripts from periodic and spectral priors)
- **Backbone**: Dual-branch Patch / Downsample encoders plus residual connection, in the spirit of LSTNN-style design
- **Spatial modeling**: GCN, dynamic graph, adaptive adjacency, hybrid graph, etc. (`ABCDSpatialModule`)
- **Prior modeling**: The 4th channel is processed by a light **KAN** branch and then fused with the backbone output
- **Framework**: Built on [BasicTS](https://github.com/zezhishao/BasicTS) / [EasyTorch](https://github.com/cure-lab/EasyTorch)

## Data Preparation

### 1. Raw data

Place dataset files under `datasets/<DATASET_NAME>/`, for example:

- **PEMS04 / PEMS07 / PEMS08**: `*.npz` (with a `data` array) plus adjacency files
- **PEMS-BAY**: `*.hdf` (pandas-readable)

### 2. Adjacency matrix (PEMS04/07/08)

```bash
cd scripts/data_preparation/PEMS08   # or PEMS04, PEMS07
python generate_adj_mx.py            # generates adj_mx.pkl
```

### 3. Train/val/test split and Prior

- **Flow + TOD + DOW only (3 channels)**: use `generate_training_data.py`
- **With Prior (4 channels, recommended for KASA)**: use `generate_holost_data.py` to produce train/val/test with Prior and normalization metadata

Example (PEMS08):

```bash
cd scripts/data_preparation/PEMS08
python generate_holost_data.py --data_file_path <path_to_npz> --graph_file_path <path_to_adj> ...
```

For other datasets, see the scripts and arguments in the corresponding subfolders under `scripts/data_preparation/`.

## Training and testing

Run training with a config (validation is run during training; testing is run after training):

```bash
# Single GPU
python examples/run.py --cfg examples/KASAST_v2/KASAST_PEMS08.py --gpus 0

# Multiple GPUs (per EasyTorch convention)
python examples/run.py --cfg examples/KASAST_v2/KASAST_PEMS08.py --gpus 0,1
```

Other datasets:

```bash
python examples/run.py --cfg examples/KASAST_v2/KASAST_PEMS04.py --gpus 0
python examples/run.py --cfg examples/KASAST_v2/KASAST_PEMS07.py --gpus 0
python examples/run.py --cfg examples/KASAST_v2/KASAST_PEMS-BAY.py --gpus 0
```

You can change `CFG.DATASET_NAME`, input/output length, batch size, learning rate, checkpoint path, etc. in the config files.

## Configuration

Main KASA-related fields in `examples/KASAST_v2/KASAST_PEMS08.py`:

- **Input/output**: `CFG.DATASET_INPUT_LEN`, `CFG.DATASET_OUTPUT_LEN` (e.g. 12/12)
- **Features**: `CFG.MODEL.FORWARD_FEATURES = [0,1,2,3]` (Flow, TOD, DOW, Prior), `TARGET_FEATURES = [0]`
- **Model**: `CFG.MODEL.ARCH = KASA_v2`; in `CFG.MODEL.PARAM` you can tune `patch_len`, `stride`, `d_*`, `use_gcn`, `use_dynamic_spatial`, `use_adaptive_adj`, `use_hybrid_graph`, etc.
- **Training**: `CFG.TRAIN.NUM_EPOCHS`, `CFG.TRAIN.OPTIM`, `CFG.TRAIN.LR_SCHEDULER`, `CFG.TRAIN.CKPT_SAVE_DIR`, etc.

For 3-channel setup (no Prior), do not generate Prior in the data scripts and set `FORWARD_FEATURES` to `[0,1,2]` and `input_dim` to 3 in the config.
